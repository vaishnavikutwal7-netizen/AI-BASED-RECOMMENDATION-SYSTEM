This project was developed as part of an internship assignment to demonstrate the fundamental working of a recommendation system using Java. The objective of this task was to design and implement a simple product recommendation application that suggests products to users based on their preferred category.

Recommendation systems are widely used in modern digital platforms such as e-commerce websites, streaming services, online learning platforms, and social media applications. These systems analyze user preferences and provide personalized suggestions to enhance user experience. This project introduces the basic concept of recommendation logic using core Java programming principles.

Objective of the Task

The main objectives of this project are:

To understand the basic concept of recommendation systems.

To implement category-based filtering logic.

To work with object-oriented programming concepts.

To store and manage product data using collections.

To take dynamic user input and generate personalized output.

This task focuses on building a simple rule-based recommendation system using Java collections and object-oriented design.

Project Overview

The application maintains a predefined list of products. Each product contains the following attributes:

Product ID

Product Name

Product Category

These products are stored in an ArrayList, which allows dynamic storage and easy iteration. The program prompts the user to enter their preferred category. Based on the input provided, the system filters the product list and displays products that match the selected category.

The filtering process uses a case-insensitive comparison to ensure flexibility and user convenience.

Functional Flow

A list of product objects is created and stored in an ArrayList.

The program asks the user to enter their preferred product category.

The system iterates through the product list.

If a product’s category matches the user’s input, it is displayed as a recommendation.

The matching products are printed in a structured format.

If no product matches the given category, the system simply does not display any result, demonstrating conditional filtering logic.
